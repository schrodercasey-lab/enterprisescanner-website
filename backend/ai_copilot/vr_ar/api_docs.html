Welcome to JUPITER VR API

The JUPITER VR API enables third-party developers to integrate with our advanced threat visualization and virtual reality security operations platform.

Key Features

  • 🔒 Secure Authentication: API keys and OAuth2 support
  • 📊 Real-time Data: WebSocket streaming for live threat updates
  • 🎮 VR Integration: Create and manage VR security sessions
  • 📈 Analytics: Access security metrics and insights
  • 🔧 Client SDKs: Python, JavaScript, Unity C#
  • ⚡ High Performance: Up to 100K requests/hour (Enterprise tier)

Base URL

http://localhost:5012

Quick Start

1. Create an API key via the portal or API

2. Use your key ID and secret in requests

3. Start making API calls!

Authentication

All API requests require authentication using API keys sent in request headers.

API Key Authentication

X-API-Key-ID: jptr_your_key_id_here
X-API-Key-Secret: your_secret_key_here

Creating an API Key

POST /api/keys
Content-Type: application/json

{
  "name": "My Application",
  "owner_id": "developer_001",
  "permissions": ["READ_THREATS", "WRITE_VR_SESSIONS"],
  "tier": "PROFESSIONAL",
  "expires_in_days": 365
}

API Permissions

Permission Description
READ_THREATS Read threat data
WRITE_THREATS Create and update threats
READ_VR_SESSIONS Read VR session data
WRITE_VR_SESSIONS Create and manage VR sessions
READ_ANALYTICS Access analytics data
ADMIN Full administrative access

REST API Endpoints

GET /api/health
Check API server health and status
GET /api/threats
Get threat data (requires READ_THREATS permission)
Query Parameters:
  - limit: Number of threats to return (default: 100)
  - severity: Filter by severity (high, medium, low)
POST /api/vr/sessions
Create a new VR session (requires WRITE_VR_SESSIONS permission)
{
  "user_id": "user_001",
  "environment": "threat_visualization",
  "config": {
    "visualization_mode": "3d",
    "interaction_enabled": true
  }
}
GET /api/vr/sessions/{session_id}
Get VR session details
GET /api/analytics/{metric}
Get analytics data for a specific metric
Query Parameters:
  - time_range: Time range (1h, 24h, 7d, 30d)
GET /api/usage
Get your API usage statistics
POST /api/webhooks
Register a webhook for event notifications
{
  "url": "https://example.com/webhooks/jupiter",
  "events": ["threat.detected", "vr.session.started"]
}

WebSocket Streaming

Connect to WebSocket endpoint for real-time data streaming.

Connection

ws://localhost:5012/ws

Authentication

{
  "type": "authenticate",
  "api_key_id": "jptr_your_key_id",
  "api_key_secret": "your_secret"
}

Subscribe to Channels

{
  "type": "subscribe",
  "channels": ["threats", "vr_sessions", "analytics"]
}

Stream Events

Event Description
threat_update New threat detected
vr_session_update VR session state changed
system_status System health update

Client SDKs

🐍 Python SDK

from jupiter_vr_client import JupiterVRClient

client = JupiterVRClient(
    api_key_id="jptr_your_key_id",
    api_key_secret="your_secret"
)

# Get threats
threats = client.get_threats(limit=50)

# Create VR session
session = client.create_vr_session({
    "user_id": "user_001",
    "environment": "threat_visualization"
})

print(f"Session ID: {session['session_id']}")

📜 JavaScript SDK

const client = new JupiterVRClient(
    'jptr_your_key_id',
    'your_secret'
);

// Get threats
const threats = await client.getThreats(50);

// Connect to WebSocket
const ws = client.connectWebSocket((data) => {
    console.log('Received:', data);
});

// Create VR session
const session = await client.createVRSession({
    user_id: 'user_001',
    environment: 'threat_visualization'
});

🎮 Unity C# SDK

JupiterVRClient client = gameObject.AddComponent<JupiterVRClient>();
client.Initialize("jptr_your_key_id", "your_secret");

// Get threats
StartCoroutine(client.GetThreats((response) => {
    Debug.Log("Threats: " + response);
}));

// Create VR session
StartCoroutine(client.CreateVRSession(configJson, (response) => {
    Debug.Log("Session created: " + response);
}));

Quota Exceeded

When limits are exceeded, you'll receive a 429 error:

{
  "error": "rate_limit_exceeded",
  "message": "Rate limit exceeded. Please wait before making more requests.",
  "retry_after_seconds": 3600
}
